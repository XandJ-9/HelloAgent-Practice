# AI Coding 学习指南

资料来源：飞书 Wiki《从零开始构建智能体》[Hello, Agents]

适合对象：已经会用对话式 vibe coding，但希望补齐 AI Coding 原理、工程流程、Agent 开发方法和完整项目实践的初学者。

## 一、文档内容总结

这份文档本质上是一套系统化的 Agent 开发教程，目标是把学习者从“大模型使用者”带到“智能体系统构建者”。它不是单纯讲提示词，也不是只教某个框架 API，而是按“基础理论 -> 单体 Agent -> 高级能力 -> 综合项目 -> 毕业设计”的路径组织。

### 1. 基础篇：理解 Agent 和 LLM

前 3 章建立概念地基：

- 智能体是什么：能感知环境、做出行动、围绕目标自主决策的系统。
- 传统智能体类型：反射式、基于模型、基于目标、基于效用、学习型智能体。
- LLM Agent 新范式：用大语言模型作为推理与规划核心，通过工具调用、记忆、反馈循环完成复杂任务。
- Agent Loop：感知、思考、行动、观察的循环。
- Workflow 与 Agent 的区别：Workflow 更像固定流程自动化，Agent 更强调动态规划、自主决策和环境反馈。
- LLM 基础：N-gram、RNN、Transformer、Decoder-Only 架构、提示工程、分词、开源模型调用、模型幻觉和缩放法则。

对你来说，这部分的重点不是深究算法，而是建立判断力：什么时候只是自动化流程，什么时候真的需要 Agent。

### 2. 单体 Agent 篇：亲手构建核心范式

第 4 到第 7 章进入实战：

- ReAct：让模型按 Thought、Action、Observation 循环思考和使用工具。
- Plan-and-Solve：先规划任务步骤，再逐步执行。
- Reflection：让 Agent 对自己的答案或代码做审查、反馈和迭代。
- 低代码平台：Coze、Dify、n8n，适合快速搭建工作流型 AI 应用。
- Agent 框架：AutoGen、AgentScope、CAMEL、LangGraph，分别展示多 Agent 协作、角色扮演、状态图编排等方式。
- 自建 HelloAgents 框架：抽象 LLM、Message、Config、Agent 基类、工具系统，并实现 SimpleAgent、ReActAgent、ReflectionAgent、PlanAndSolveAgent、FunctionCallAgent。

这一部分是从 vibe coding 迈向专业开发的核心。你会开始理解：AI Coding 不是“把需求扔给模型”，而是把任务拆成状态、工具、约束、反馈、评估和可维护代码。

### 3. 高级篇：让 Agent 更像长期工程系统

第 8 到第 12 章补齐高级能力：

- 记忆与 RAG：让 Agent 能保存用户偏好、历史上下文、外部知识，并基于检索回答。
- 上下文工程：学习如何组织有效上下文，而不是盲目把所有材料塞进 prompt。
- NoteTool 和 TerminalTool：让 Agent 能维护结构化笔记、访问文件系统、处理长程任务。
- 通信协议：MCP、A2A、ANP，用标准协议连接工具、服务和其他智能体。
- Agentic RL：介绍从 SFT 到 GRPO 的训练流程，理解数据集、奖励函数、评估和部署。
- 性能评估：使用 BFCL、GAIA、LLM Judge、Win Rate 等方法评估工具调用、通用助手能力和生成质量。

这一部分非常接近真实 AI 工程：上下文怎么管、工具怎么接、系统怎么评估、结果怎么改进。

### 4. 综合项目篇：把能力串成完整应用

第 13 到第 15 章是三个完整案例：

- 智能旅行助手：多 Agent 协作、MCP 工具集成、Pydantic 数据模型、FastAPI 后端、前端页面和导出功能。
- 自动化深度研究智能体：TODO 驱动研究、搜索工具、笔记工具、任务规划、总结服务、研究报告生成。
- 赛博小镇：NPC Agent、角色设定、记忆系统、好感度系统、FastAPI 后端、Godot 游戏前端、前后端通信。

这部分最适合练完整开发流程：需求分析、架构设计、数据建模、后端服务、前端交互、Agent 行为设计、测试与迭代。

### 5. 毕业设计篇：做自己的多 Agent 应用

第 16 章讲如何完成一个毕业项目：

- 选题原则和推荐方向。
- 环境准备、Fork 仓库、分支开发。
- 编写 README、requirements.txt、Jupyter Notebook。
- 测试项目、处理大文件、提交 Pull Request、响应 Review。
- 示例项目：智能代码审查助手。

这部分对你最重要的启发是：学习 AI Coding 的终点应该是可运行、可说明、可复现、可评估的项目，而不是一堆零散聊天记录。

## 二、推荐学习大纲

建议用 8 周完成第一轮。目标不是把所有内容背下来，而是每周做出一个可运行的小成果。

### 第 0 周：整理学习目录

目标：把当前目录变成可持续学习空间。

建议结构：

```text
HelloAgent-Practise/
  AI_CODING_LEARNING_GUIDE.md
  notes/
  labs/
  projects/
  prompts/
  reviews/
```

你每学一章，都至少留下三类产物：一份笔记、一个可运行练习、一次复盘。

### 第 1 周：从 vibe coding 到工程化 AI Coding

学习重点：

- Agent、Workflow、LLM App 的区别。
- Agent Loop：感知、思考、行动、观察。
- AI Coding 的完整流程：需求澄清、方案设计、任务拆解、实现、运行、调试、测试、复盘。

练习：

- 写一份 `notes/01-agent-basics.md`，用自己的话解释 Agent。
- 用 Python 写一个最小 Agent Loop：输入任务、选择工具、执行工具、返回结果。
- 记录一次你平时 vibe coding 的过程，并改写成“专业开发流程清单”。

### 第 2 周：补齐 LLM 与 Prompt 基础

学习重点：

- Transformer 和 Decoder-Only 的基本直觉。
- Token、上下文窗口、模型幻觉、模型选择。
- Prompt 的角色、任务、约束、示例、输出格式。

练习：

- 建立 `prompts/` 目录，保存 5 个常用开发提示词：需求澄清、代码生成、代码审查、测试生成、重构建议。
- 写一个脚本调用 LLM API，完成“需求 -> 技术方案”的生成。
- 对同一任务比较 3 种 prompt 写法的输出差异。

### 第 3 周：掌握 ReAct 和工具调用

学习重点：

- ReAct 的 Thought、Action、Observation 循环。
- 工具定义、工具注册、参数校验、工具返回值。
- 工具调用失败时如何恢复。

练习：

- 在 `labs/react_agent/` 中实现一个带工具的 ReAct Agent。
- 工具至少包含：计算器、文件读取、搜索或模拟搜索。
- 给 Agent 加日志，记录每一步 Thought、Action、Observation。

验收标准：

- 能解释 Agent 为什么选择某个工具。
- 工具报错时不会直接崩溃，而是能把错误反馈给模型或用户。

### 第 4 周：Plan-and-Solve 与 Reflection

学习重点：

- 复杂任务先规划再执行。
- Reflection 用于代码审查、答案修正、测试失败修复。
- 多轮迭代的成本与收益。

练习：

- 实现一个“代码修改助手”：先生成计划，再按步骤修改一个小项目。
- 实现一个 Reflection 流程：生成代码、运行测试、根据错误反思、再次修复。
- 把每次迭代结果写入 `reviews/`。

验收标准：

- 每次任务开始前有计划。
- 每次失败后有错误分析，而不是盲目重试。

### 第 5 周：学习框架，但不被框架绑架

学习重点：

- AutoGen、CAMEL、AgentScope、LangGraph 的定位。
- 状态图、角色协作、多 Agent 对话。
- 什么时候用框架，什么时候手写更清楚。

练习：

- 用 LangGraph 或你喜欢的框架做一个“三步问答助手”：规划、检索、回答。
- 画出状态流转图。
- 写一份对比笔记：手写 Agent 与框架 Agent 的优缺点。

### 第 6 周：RAG、记忆与上下文工程

学习重点：

- RAG 的索引、切分、向量化、召回、重排、生成。
- 记忆类型：工作记忆、情景记忆、语义记忆、程序性记忆。
- 上下文工程：如何选择、压缩、组织和更新上下文。

练习：

- 建一个“学习文档问答助手”，把你的笔记作为知识库。
- 让它能回答“我上周学了什么”“ReAct 和 Plan-and-Solve 有什么区别”。
- 为每次回答输出引用来源或依据。

验收标准：

- 不知道时能承认不知道。
- 回答能追溯到具体笔记或文档片段。

### 第 7 周：AI Coding 的评估、测试与安全

学习重点：

- 单元测试、集成测试、端到端测试。
- Agent 评估：任务成功率、工具调用准确率、成本、延迟、稳定性。
- 安全边界：密钥管理、文件权限、危险操作确认、用户数据保护。

练习：

- 给前几周的 Agent 加测试。
- 设计一个简单评估集，至少 20 条任务。
- 统计成功率、失败原因、平均耗时。

验收标准：

- 你能说清楚当前 Agent 哪些任务可靠，哪些任务不可靠。
- 所有密钥放在 `.env`，不写进代码或笔记。

### 第 8 周：毕业小项目

推荐选题三选一：

- 智能代码审查助手：读取项目文件，指出 bug、风险、缺失测试，并生成修复建议。
- 个人学习研究助手：根据主题自动拆 TODO、搜索资料、做笔记、生成报告。
- 文档问答助手：对你的学习目录进行 RAG 检索，回答学习问题并引用来源。

完整流程：

1. 写需求文档：目标用户、核心场景、输入输出、成功标准。
2. 画架构图：LLM、工具、存储、前端或命令行入口。
3. 做最小可用版本。
4. 加日志、错误处理、测试和 README。
5. 做一次复盘：哪些是模型能力问题，哪些是工程设计问题。

## 三、你应该形成的专业工作流

以后做 AI Coding，不建议直接一句话让模型“帮我做个项目”。可以按这个顺序来：

1. 需求澄清：我要解决什么问题，给谁用，输入输出是什么。
2. 约束确认：技术栈、运行环境、时间、成本、安全边界。
3. 方案设计：模块、数据流、工具、状态、错误处理。
4. 任务拆解：每次只让 AI 处理一个明确的小任务。
5. 实现验证：运行、测试、截图或日志验证。
6. 代码审查：检查边界条件、异常、安全、可维护性。
7. 复盘沉淀：把提示词、坑点、失败案例和最终方案留下来。

## 四、第一轮学习目标

完成第一轮后，你应该能做到：

- 看懂一个 Agent 项目的基本结构。
- 自己实现一个带工具调用的简单 Agent。
- 用计划、执行、反思的方式完成代码任务。
- 能判断什么时候用低代码平台、什么时候用 Agent 框架、什么时候手写。
- 能把 AI 生成代码纳入测试、评估、复盘流程。
- 拥有一个可展示的小型 AI Coding 项目。
